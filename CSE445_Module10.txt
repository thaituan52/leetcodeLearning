Google Protocol Buffers and BigTable
1. Motivation
Google requires extreme efficiency in data representation, transfer, and storage to handle massive scale.

2. Protocol Buffers (protobufs)


Purpose: Efficient data serialization format developed by Google.


Advantages over XML:


3–10× smaller in size.


20–100× faster to parse.


Less ambiguous; machine-friendly.


Generates code (classes) for many languages (C++, Java, Python).




Drawback: Less flexible; schema changes need regeneration.


Example:
message Person {
  required string name = 1;
  required int32 id = 2;
  optional string email = 3;
}



Use in gRPC:


gRPC = Google’s open-source RPC framework.


Uses protobuf as Interface Definition Language (IDL).


Enables direct remote method calls (cross-language).





3. Google BigTable


Purpose: Large-scale, distributed, non-relational database system.


Built On: Google File System (GFS).


Structure: “Sparse, distributed, multi-dimensional sorted map.”


Design Concepts:


Inspired by B+ Trees for efficient key-based indexing.


Data organized as key–value pairs, sorted for fast access.


Scales horizontally across thousands of machines.





4. B-Trees and B+ Trees


B-Trees: Balanced search trees storing keys and values.


B+ Trees: Variant where all data (values) stored in leaf nodes, enabling efficient range queries.


Used in BigTable to manage keys and locate data across distributed servers.
